## Three-Ways-to-Start Decision Tree

```
Blank project → No ideas?
  ├── Need structural clarity? → Start with foundation (bass + drums)
  ├── Have spontaneous ideas? → Start with what you hear (hum, tap, record)
  ├── Play physical instrument? → Start with what you know (guitar, drums)
  └── Nothing at all? → Goal-less exploration (ch15) or arbitrary constraints (ch06)
```

## Write Drunk; Edit Sober Workflow

```
Phase 1: CREATION
  ├── Always recording
  ├── Delete nothing
  ├── Judge nothing
  ├── Aim for speed and quantity
  └── Follow instinct

Phase 2: EDITING
  ├── No new material
  ├── Ruthlessly curate
  ├── Delete, trim, reorder
  └── Expect to throw away most of Phase 1
```

## Harmony Quick Reference

### Major Scale: W-W-H-W-W-W-H
### Minor Scale: W-H-W-W-H-W-W

### Diatonic Triads (Major Key)
| Degree | Chord | Function |
|--------|-------|----------|
| I | Major | Tonic (home) |
| ii | Minor | Pre-dominant |
| iii | Minor | Substitute for I |
| IV | Major | Pre-dominant |
| V | Major | Dominant (tension) |
| vi | Minor | Substitute for IV |
| vii° | Diminished | Dominant substitute |

### Most Common Progressions
- I-V-vi-IV (the "pop progression")
- I→anything (I is home base)
- V/vii°→I (dominant→tonic)
- ii/IV→V (pre-dominant→dominant)
- vi→ii/IV (substitute for IV)

## Arrangement Decision Guide

```
Need section contrast?
  ├── Electronic genre? → Layering as form (buildup/breakdown/drop)
  ├── Vocal song? → ABABCB form (verse/chorus/bridge)
  └── Unsure? → Copy reference structure (formal skeleton)

Music feels weak/cluttered?
  ├── Take parts away, don't add
  ├── Thin out elements (fewer notes, shorter decays)
  └── Each added part must earn its place

Need an ending?
  ├── Mirror what you did at the start
  ├── Loop closing material, add/subtract layers each repeat
  └── Fade out (only if closing material works looping indefinitely)
```

## Drum Programming Quick Reference

### Quantization Approach
| Technique | When to Use |
|-----------|-------------|
| Full quantization | Tight, precise beats (techno, electronic) |
| Partial quantization | Human-feeling but controlled |
| Selective quantization | Kick/snare tight, hi-hats loose |
| No quantization | Loose, organic feel |

### Behind vs. Ahead of Beat
| Position | Feel | Genre |
|----------|------|-------|
| Behind the beat | Laid-back, relaxed | Hip-hop, R&B, reggae |
| On the beat | Direct, in-your-face | Rock, punk |
| Ahead of the beat | Driving, urgent | Drum & bass, breakbeat |

### Linear Drumming Checklist
- [ ] No two drum instruments play simultaneously
- [ ] Each drum hit uses a different instrument
- [ ] Treat drum part as a quasi-melodic line
- [ ] Each instrument equally weighted

### Kick + Bass Composite Checklist
- [ ] Zero overlap of attacks
- [ ] Bass notes fit BETWEEN kick notes
- [ ] Sustained bass notes OK if onsets don't overlap
- [ ] Treat as ONE monophonic line, not two

## Problem-Solving Decision Tree

```
Technical problem during creative time?
  ├── Can this wait? → Yes → Note it, return to creation
  ├── Can this wait? → No → It's a real problem, solve it
  └── Unsure? → Assume it can wait (Resistance is common)
```

## Finishing Checklist

- [ ] Recognized "good enough" (not chasing perfection)
- [ ] Used subtractive arranging (took away, didn't add)
- [ ] Applied dramatic arc (exposition → climax → dénouement)
- [ ] Chose deliberate ending (mirror, loop+modify, or fade)
- [ ] Rendered to audio (commitment device)
- [ ] Got feedback from appropriate sources
- [ ] Saved as scrap for future reuse (even if bad)

## Randomization Evaluation Protocol

1. Generate random output
2. Solo the result
3. Slow it down to assess
4. Bounce to audio to evaluate
5. Increase randomization until too extreme, then dial back
6. Take ownership of every result — you chose the parameters

## Listening Evaluation Checklist

- [ ] Active listening (primary activity, not background)
- [ ] Multiple passes (sound, harmony, melody, rhythm, form)
- [ ] Loop small chunks for deep analysis
- [ ] Listen subjectively (understand emotional responses)
- [ ] Listen in bad conditions (corners, other rooms, extreme volume)
- [ ] Listen with serious fans/other producers for feedback
