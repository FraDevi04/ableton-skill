# Chapter 2: Catalog of Attributes

## Core Idea
Extract the structural essence of inspiring music into a written catalog, then use that catalog as a template for creating entirely new work — capturing influence without plagiarism.

## Frameworks Introduced
- **Catalog of Attributes**: A structured description of an inspiring piece's sound, harmony, melody, rhythm, and form. Write it down, set aside the original, and create new music using only the catalog as a guide.
- **Template-based creation**: The catalog is complete enough to serve as a recipe but not specific enough to allow recreation — two musicians reading the same catalog would create completely different pieces.

## Key Concepts
- **Homage vs. plagiarism**: The boundary is subjective; the catalog method lets you infuse essence without copying
- **Attribute categories**: Sound, harmony, melody, rhythm, form — analyze each element separately
- **Framework vs. transcription**: Capture scaffolding, not exact notes or sounds
- **Level of detail**: Varies by your ability to describe, complexity of source, and time invested — specificity matters less than usability

## Mental Models
- Use the catalog when you want to be influenced by specific music without directly copying it
- Think of the catalog as a "recipe" that produces varied results — it's a constraint system, not a blueprint
- The catalog forces you to analyze music actively rather than just absorbing it passively

## Anti-patterns
- **Transcribing instead of cataloging**: Writing exact notes defeats the purpose
- **Keeping the original nearby**: You must set it aside and work only from the catalog
- **Over-describing**: If the catalog is too specific, it allows recreation rather than inspiration

## Key Takeaways
1. Listen carefully and many times to the source before writing the catalog
2. Study element by element — sound, harmony, melody, rhythm, form
3. Write descriptions, not transcriptions — capture the framework
4. Set aside the original once the catalog is complete
5. Use the catalog as a template — two musicians would create different music from the same catalog
