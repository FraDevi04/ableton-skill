# Chapter 28: Creating Variation 1: Mutation of Clones

## Core Idea
Duplicate an idea 8 times, then make one meaningful change to each copy. This creates a family of related variations from a single source.

## Frameworks Introduced
- **Mutation of clones**: Duplicate an idea 8 times, make one meaningful change to each copy. Each copy is related to the original but distinct.
- **One-change principle**: Each variation has exactly ONE meaningful change — this keeps the family coherent while providing variety.
- **Family coherence**: All variations share a common DNA, making them recognizable as related.

## Key Concepts
- **Variation through subtraction/addition**: Remove one element, add one element, or change one parameter
- **Meaningful change**: The change must be musically significant — not just a random tweak
- **Eight-copy limit**: 8 copies provides enough variety without becoming overwhelming
- **Common DNA**: All variations must share a recognizable element from the original

## Mental Models
- Use mutation of clones when you need variations on a theme
- Think of it as "evolution from a single ancestor"
- Each copy should be recognizable as related to the original

## Anti-patterns
- **Multiple changes per copy**: More than one change per copy loses the "mutation" clarity
- **Meaningless changes**: Changing volume by 1 dB is not a meaningful change
- **Too many or too few copies**: 8 is the sweet spot — more creates confusion, less creates boredom

## Key Takeaways
1. Duplicate an idea 8 times, make one meaningful change to each copy
2. Each variation has exactly ONE meaningful change
3. All variations share a common DNA from the original
4. The change must be musically significant — not just a random tweak
5. This creates a family of related variations that provide variety without losing coherence
