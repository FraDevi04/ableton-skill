# Chapter 12: The Tyranny of the Default

## Core Idea
Build custom template projects that override your DAW's defaults, saving time and capturing musical ideas before motivation fades.

## Frameworks Introduced
- **Tyranny of the Default**: Software developers' decisions affect all users who never change defaults. Your DAW's default project is designed to offend no one but optimize for no one.
- **Custom templates**: Save projects with your preferred tempo, instruments, effects, and settings. Create multiple templates for different genres or workflows.
- **Pre-loaded instruments**: Arm tracks for recording, preload go-to presets, create default tracks with your常用 effects chain.

## Key Concepts
- **Motivation decay**: By the time you've set up your DAW, you've lost the idea or motivation
- **Genre-appropriate defaults**: If you work at 128 bpm, don't start from 120 bpm default
- **Multiple templates**: Different genres = different templates. Save As to preserve each.
- **Construction kit danger**: Using pre-made templates with finished songs borders on plagiarism

## Mental Models
- Use custom templates to eliminate setup time and capture ideas faster
- Think of templates as your personal DAW configuration, not blank slates
- Create templates for different scenarios (sketching, collaboration, specific genres)

## Anti-patterns
- **Using DAW defaults**: Every new project starts from someone else's assumptions
- **One template for everything**: Different genres need different default configurations
- **Construction kit dependency**: Using near-finished templates as your starting point

## Key Takeaways
1. Build custom template projects with your preferred settings
2. Set genre-appropriate tempo, sample rate, bit depth, and layout
3. Pre-load常用 instruments and effects — make them ready to use immediately
4. Create multiple templates for different genres or workflow types
5. Be careful not to use pre-made templates as construction kits for your own music
